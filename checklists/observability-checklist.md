# Observability checklist

- [ ] Structured logs from application  
- [ ] Request metrics (rate, errors, duration)  
- [ ] Host or platform metrics (CPU, mem, disk)  
- [ ] DB metrics (connections, CPU, storage)  
- [ ] Dashboards linked from runbooks  
- [ ] Alerts have owners and runbook links  
- [ ] Tracing (if multi-service) sampled appropriately  

Noise control: if alert fires weekly without action, fix threshold or remove.
