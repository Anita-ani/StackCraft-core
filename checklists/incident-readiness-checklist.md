# Incident readiness checklist

- [ ] Severity definitions (Sev1–3) agreed  
- [ ] On-call rotation or explicit “best effort” owner  
- [ ] Internal comms channel for incidents  
- [ ] Customer status page or template for external updates  
- [ ] Rollback runbook linked from deploy docs  
- [ ] Postmortem template ready ([../templates/postmortem-template.md](../templates/postmortem-template.md))  
- [ ] Postmortems blameless; action items tracked
